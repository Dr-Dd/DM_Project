\maketitle

In this project, I will attempt to qualitatively compare two different
data models and their associated technologies. Specifically, the
comparison will include the Resource Description Framework and the
Relational Model. It will be conducted in the following manner:

\begin{enumerate}
  \item Choosing a dataset and an associated real-world scenario
  related to its use.
  \item Creating two implementations of the dataset, based on the
  requirements outlined in the previous steps.
  \item Importing the respective implementations and the dataset into
  a relational database and a triplestore.
  \item Comparing various aspects of the two solutions, both practical
  and conceptual (e.g. query language, memory usage, query execution
  speed, model flexibility in representing the dataset.).
\end{enumerate}
For this purpose, the chosen dataset, RDBMS and triplestore are:

\begin{itemize}
  \item Dataset(s): \href{https://datasets.imdbws.com/}{The IMDb
    Non-Commercial Datasets}, for their size, ease of access and
  simple approachability, which contains the full IMDb website with
  all its information regarding movies, actors, and their relations.
  \item RDBMS: \href{https://www.postgresql.org/}{PostgreSQL}, being
  open-source, mature and on-par with current industry standards.
  \item RDF Store (triplestore):
  \href{https://jena.apache.org/documentation/fuseki2/}{Apache Jena
    Fuseki}, for the same reasons as Postgres.
\end{itemize}
